# Profile

**Document Version:** 1.0  
**Status:** Draft  
**Last Updated:** July 2026

---

# 1. Purpose

The Profile is the user's personal space within the platform.

It provides identity information, learning statistics, achievements in terms of progress, and personalization settings.

The Profile should help users understand how far they have progressed in their learning journey.

---

# 2. Goals

The Profile should allow users to:

- manage personal information;
- customize their identity;
- review learning progress;
- track historical performance;
- share a public profile.

The Profile should emphasize measurable progress rather than competition.

---

# 3. Profile Information

Each user profile contains:

- Avatar
- Display Name
- Username
- Short Bio (optional)
- Registration Date
- Preferred Language

The email address is private and is never displayed publicly.

---

# 4. Avatar

Users may choose between:

- predefined avatars;
- uploading a custom avatar.

Only one avatar may be active at any time.

Supported image formats:

- PNG
- JPG
- WEBP

---

# 5. Public Profile

Each user has a public profile.

Public information includes:

- Avatar
- Display Name
- Username
- Current Level
- Total XP
- Completed Quizzes
- Average Accuracy
- Favorite Subject (future)

Private information is never visible.

---

# 6. Progress Overview

The Profile displays:

- Current Level
- Total XP
- XP Progress to Next Level
- Total Quiz Sessions
- Questions Answered
- Average Accuracy
- Total Learning Time

This section should immediately communicate the user's learning progress.

---

# 7. Statistics

The Profile provides access to detailed statistics, including:

- Quizzes Completed
- Questions Answered
- Correct Answers
- Incorrect Answers
- Average Accuracy
- Best Accuracy
- Total XP Earned

Detailed analytics may expand in future versions.

---

# 8. Activity History

Users can review recent quiz sessions.

Each entry includes:

- Subject
- Quiz Score
- Accuracy
- XP Earned
- Completion Date
- Duration

The newest sessions appear first.

---

# 9. Personalization

Users can customize:

- Avatar
- Display Name
- Username
- Bio
- Language
- Theme

Future versions may include additional personalization options.

---

# 10. Privacy

The platform distinguishes between public and private information.

### Public

- Avatar
- Display Name
- Username
- Current Level
- Total XP

### Private

- Email Address
- Password
- Authentication Data
- Account Settings

Sensitive information is never exposed to other users.

---

# 11. Validation

Display Name:

- required;
- maximum 50 characters.

Username:

- unique;
- 3–30 characters;
- letters, numbers, underscores.

Bio:

- optional;
- maximum 250 characters.

Avatar uploads must pass image validation.

---

# 12. Empty States

New users may have little or no activity.

Instead of empty sections, the Profile should display encouraging messages.

Example:

> Complete your first quiz to start building your learning history.

---

# 13. Future Improvements

Potential future additions include:

- Favorite Subjects
- Learning Goals
- Weekly Progress
- Monthly Reports
- Learning Calendar
- Export Learning Statistics
- Social Links

These features are outside the scope of the MVP.

---

# 14. Non-Functional Requirements

The Profile should:

- load quickly;
- remain responsive on all devices;
- present information clearly;
- support future expansion without redesign.

---

# 15. Success Criteria

The Profile is considered successful if users can:

- recognize their identity within the platform;
- understand their learning progress;
- review previous activity;
- personalize their account;
- feel motivated to continue learning.

The Profile should reinforce the platform's core philosophy:

> Learn. Progress. Repeat.
